Big-picture idea (intuition)
Sliding window is a technique for problems about contiguous sequences (subarrays or substrings).
Maintain a window [left, right) or [L, R] that “slides” over the array/string 
while maintaining some property (sum, count, max, freq).


Two main flavors:
Fixed-size window: window size is predetermined (k). 
Common tasks: sum, max in every window of size k.

Variable-size window: window expands/contracts to satisfy constraints
(e.g., smallest window with sum >= S, or longest window with <= K distinct).

Complexity: sliding window usually gives O(n) or O(n log n) depending on the
data structure (deque, hashmap, balanced tree).


When to use sliding window — pattern checklist (how to guess) Ask these questions:

Does the problem ask about subarray or substring (contiguous)? 
If yes → candidate for sliding window.

Are you asked for longest/shortest/maximum/minimum/count of contiguous subarrays/substring?
→ likely sliding window.

Does the window’s quality depend on a running aggregate
(sum, product, count distinct, freq)? → yes.

Is there no reordering allowed, or the sequence must be contiguous? → sliding window fits.
Constraints hint: expected O(n) or O(n log n). If n up to 1e5 or higher and optimal is O(n), 
sliding window is typical. When NOT to use sliding window:

The subsequence is non-contiguous or can be re-ordered
(then consider two-sum, sorting, DP, prefix sums).


The window property requires global info that can't be updated in O(1)
or amortized O(1) (unless you use a supporting structure).

There are negative numbers and the property depends on arbitrary sums allowing shrinking without monotonicity (minimum-length with sum >= target with negative numbers is harder).
Common sliding-window problem types (and core ideas)
Fixed-size window tasks:
Maximum/average/sum of every window of size k → maintain running sum or deque for max.
Variable-size window (two pointers) tasks:
Longest substring without repeating characters → expand until duplicate, then shrink until valid.
Minimum size subarray sum >= S → expand right to reach S, then shrink left to try smaller windows.
At most K distinct characters/elements → maintain freq map and shrink when > K.
Count subarrays with sum equal to K (non-negative numbers → sliding window; negative allowed → prefix sum + hashmap)
Monotonic-queue variations:
Sliding window maximum/minimum → use deque storing indices in decreasing order.
Subarray with max-min <= limit → maintain two deques (max and min).
Frequency/anagram checks:
Check if an anagram of pattern exists as substring → maintain counts in window.
Count occurrences of permutations → sliding window with frequency compare.
Templates (pseudocode + ready-to-use code) A. Fixed-size window (sum/avg/max via sum) Python template (sum):
Python
def fixed_size_window(arr, k):
    if k > len(arr): return []
    window_sum = sum(arr[:k])
    results = [window_sum]
    for i in range(k, len(arr)):
        window_sum += arr[i] - arr[i-k]
        results.append(window_sum)
    return results
Complexity: O(n).

B. Variable-size window (two pointers) — general pattern

Expand right pointer until constraint broken or satisfied.
While constraint satisfied (or violated depending on objective), update answer and shrink left. Python template:
Python
def variable_window(arr):
    left = 0
    cur_state = initial_state()
    best = initial_best()
    for right in range(len(arr)):
        add(cur_state, arr[right])
        while should_shrink(cur_state):
            # update best if objective requires
            best = update_best(best, left, right, cur_state)
            remove(cur_state, arr[left])
            left += 1
        # optionally update best after expansion
        best = update_best(best, left, right, cur_state)
    return best
Implement add/remove/should_shrink/update_best per problem.

C. Sliding Window Maximum (deque) Python:

Python
from collections import deque

def sliding_window_max(nums, k):
    dq = deque()  # store indices, nums[dq[i]] decreasing
    res = []
    for i, val in enumerate(nums):
        # pop out of window
        if dq and dq[0] <= i - k:
            dq.popleft()
        # pop smaller values
        while dq and nums[dq[-1]] < val:
            dq.pop()
        dq.append(i)
        if i >= k - 1:
            res.append(nums[dq[0]])
    return res
Complexity: O(n) amortized.

Three full example problems with explanation & Python code
Example 1 — Longest Substring Without Repeating Characters (variable window)

Problem: Given string s, find length of longest substring without repeating chars.
Idea: Expand right, track frequency / last index; when duplicate, move left past previous index. Python:
Python
def length_of_longest_unique_substring(s):
    last = {}  # char -> last index
    left = 0
    best = 0
    for right, ch in enumerate(s):
        if ch in last and last[ch] >= left:
            left = last[ch] + 1
        last[ch] = right
        best = max(best, right - left + 1)
    return best
Complexity: O(n), space O(min(n, charset)).

Example 2 — Minimum Size Subarray Sum (variable window)

Problem: smallest length subarray with sum >= target (array of positive integers).
Idea: Expand right to accumulate sum, then while sum >= target shrink left to find smaller windows. Python:
Python
def min_subarray_len(target, nums):
    left = 0
    cur = 0
    best = float('inf')
    for right, x in enumerate(nums):
        cur += x
        while cur >= target:
            best = min(best, right - left + 1)
            cur -= nums[left]
            left += 1
    return 0 if best == float('inf') else best
Complexity: O(n).

Variations & advanced techniques
Monotonic deque (max/min): keep indices in deque such that their values are monotonic. O(n).
Two deques for max and min: used to keep track of both extremes for conditions like max-min <= limit.
When elements can be negative: for some sum constraints sliding window breaks; use prefix sums + hashmap for exact sum K when negatives allowed.
When order doesn’t matter or you need combinations, sliding window often not applicable.
When counting subarrays with sum <= K for non-negative numbers you can use sliding window + counting; for negatives use prefix sums.
Multi-character constraints: for k distinct/frequency equality problems use hashmap counters and match-counters.
Complexity expectations
Basic sum/count sliding window: O(n).
Sliding window + hashmap: O(n) amortized.
Sliding window + deque: O(n) amortized.
If using balanced trees (e.g., to get median), cost becomes O(n log n) (e.g., sliding window median).
Common pitfalls and corner cases
Off-by-one on indices, choosing inclusive vs exclusive windows. Use one convention and stick with it.
For fixed k, ensure k ≤ n case.
For variable windows with negative numbers, sliding shrink condition may not be monotonic — sliding window may fail.
When using dictionary for frequencies, forget to decrement/remove keys (manage counts correctly).
For sliding window maximum, don't forget to remove indices that fell out of window (i - k).
Initialization: best = 0 or -inf vs float('inf') depending on objective.
Edge cases: empty array/string, k=0, k=1, all equal elements.
